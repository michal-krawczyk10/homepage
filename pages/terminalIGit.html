podstawy gita

repozytorium = projekt, wszystkie pliki naszego projektu

commit = stan projektu, rejestr zmian w plikach, zawiera dokładne informacje co się zmieniło od poprzedniego commita, tworzy wersję do której można wrócić
commit zapisuje historie zmian

komendy git

git --version /sprawdzanie wersji git config - konfiguracja git status git add . /dodaje pliki do śledzenia przez git. Kropka dodaje wszystkie pliki z katalogu w który, się znajdujemy, plus podkatalogi. Ważne, nie dodaje plików z katalogów równoległych ani wyższych. git commit -m "opis zmian" /tworzy commita na stan obecny wszystkich plików dodanych do śledzenia. Śledzi zmiany w plikach edytowalnych przez edytor tekstu (np. css), nie śledzi zmian w innych typach (np. jpg, czy txt), te pliki podmieni w całości. Lepiej komitować częściej, np. po stworzeniu każdej sekcji, po dodaniu klas do sekcji itd. Nie wystarczy po np. dniu pracy. gitk - podgląd historii commitów projektu
