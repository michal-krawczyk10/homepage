Własności klas i ich wartości

Stylowanie wielkości i położenia elementów elementów

display

block / elementy ustawiają się jeden pod drugim

inline / elementy ustawiają się obok siebie

inline-block / element zachowuje się jak liniowy, ale można mu nadawać własności blokowe, np width/height

width/height / nie działa w display: inline

max-width/height / height rzadko się korzysta

min-width/height

overflow / wpływa na treść wobec boxa

visible / domyślna wartość

hidden / chowa treść jeśli jest jej za dużo

auto - dodaje scroll w elemencie

box model (do zrobienia graficzka, z hooverami itp)

wszystkie własności css, które mają kierunki działają na zasadzie zegarka, wartości można podawać w kolejności top - right - bottom - left.

padding / wewnętrzny margines

padding-top/left/right/buttom: (jednostka)

border / obramowanie, też działa zasada zegarka

zapis skrótowy:
border: 10px solid red (szerokość, styl, kolor)

border-radius / zaokrąglenie granic. wartość w jednostkach, np. 20px. 50% - daje okrąg.

margin / margines, odsuwa element od innych elementów. Nie jest jego częścią. Zapis taki sam jak w paddingach. Działa tylko przy elementach blokowych. Marginesy różnych elementów się sumują

auto / jeśli element ma display: block && ustaloną szerokość, margin: __px auto wyśrodkuje element w poziomie nadając automatyczne równe marginesy po obu stronach.

box-sizing grafika/

content-box / domyślna wartość, mówi, że definiowana szerokośc/wysokość, odnosi się do kontentu, nie wpływa na padding, border i margines

border-box / wlicza do wielkości elementu, także padding i border. dzięki temu łatwiej można nadać elementowi rządany rozmiar

position

position: static /domyślna wartość

position: relative / sprawia,że element staje się domyślny dla podelementów i można je pozycjonować wg. niego za pomocą kolejnej wartości

position: absolute / pozycjonuje element względem elementu relative, na zasadzie układu współrzędnych. Jeśli nie ma elementu relative, pozycjonuje od body. Należy mu nadać kolejne własności:

top, right, bottom, left
przykład
top: __px; ustawia element o pozycji absolute o wskazaną odkległość od górnej krawędzie elementu relative.

z-index / definiuje warstwy/trzeci wymiar. Gdy elementy nachodzą na siebie, widoczny jest ten o wyższej wartości z-index

position: fixed
pozycjonuje element na stałe w jednym miejscu viewportu, niezależnie od innych elementów

position: sticky / przykleja się do wyznaczonego miejsca i pozostaje w nim dopuki parent-element jest widoczny. żeby zadziałał, rodzic nie może mieć overflow'a.

float

przyjmuje wartości jak absolut (left itd)

nie budujemy nim layoutów. służy do wstawiania elementu, który będzie "opłynięty" treścią rodzica.

clear: left, right, both, dopasowuje wielkośc rodzica, jeśli float poza niego wystaje

overflow:hidden / chowa float pod krawędzią rodzica, jeśli float chce wystawać poza

Jednostki wielkości

0 - zawsze jest zerem, można dopisać po nim dowolną jednostkę, ale nie trzeba

px - piksel

% - procenty liczone są względem elementu rodzica

vw - procent szerokości viewportu (okna)

vh - procent wysokości viewportu. używane często jest 100vh

BEM - Block Elemenent Modifier

Standard nazywania klass.
block__element--modifier
Przykład:
Podsumowując, tworząc BEM
- szukamy elementów na stronie i dzielimy ją na bloki
- myślimy o elementach składniowych / robimy analizę-selekcję
- tworzymy dwie klasy jeśli element ma modyfikator. W modyfikatrorze zmieniamy tylko unikalne dla niego wartości, nie przypisujemy rzeczy z klasy bazowej.
